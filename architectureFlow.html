GRIT Architecture

How The App Works

Grassroot Innovation In Technology connects public idea submission, core committee governance, and category-scoped jury scoring on MongoDB.

End-to-end flow

Competition Lifecycle

1. Core opens cycleCore sets one start/end window for all categories.
→
2. Users submitPublic users submit before due date and edit with private token until jury release.
→
3. Core assigns panelsCore adds/removes leads and members per category.
→
4. Release to juryScoring opens only after release; release can be withdrawn for testing.
→
5. Jury scoresMembers score. Leads see category averages, scores, and comments.
→
6. Confirm/archiveLead confirms winners. Core reviews final winners, closes jury, and archives cycle.

Explore by role

Role Responsibilities

Public accessNo login required for viewing, searching, submitting, commenting, or reacting.
Idea detailProblem, solution, rich/plain content, images, optional image names, contributors, sponsor, and categories.
Edit controlOnly the submitter with the private edit token/session can edit before cycle expiry.
FeedbackVisitors can comment and like/neutral/dislike; cached counts keep gallery loading fast.
Cycle windowOne start/end deadline applies to every category in that cycle. New submissions close at due date.
PanelsCreate, add existing, remove, and replace jury leads/members in category panels.
Core membersAdd or disable variable core committee members from Categories or Accounts.
Release controlsRelease to jury, withdraw release for testing, close jury, archive cycle.
Management viewTotal submitted ideas, counts by category, final winners, sponsor, submitter, and lead final comment.
Category scopeLead sees only assigned categories.
Peer visibilityLead sees same-category jury member scores and comments.
Final commentLead adds the overall final comment for top ideas.
Confirm winnersLead confirms top ideas based on the category target set by core.
Assigned reviewMembers score only assigned categories after release.
Scoring1-10 score, comment, and like/neutral/dislike signal.
ProgressScored and pending items are colored and counted.
PrivacyMembers do not see peer scores unless they are assigned as lead.
CollectionStores
usersCore, jury lead, and jury member accounts with hashed passwords.
cyclesCycle name, single deadline, release/withdraw/close/archive state.
categoriesCategory names, winner target count, assigned jury lead/member IDs.
ideasSubmission content, edit token, contributors, image metadata, public comments, cached reactions.
evaluationsCategory-scoped jury score, comment, and signal.
idea_reactionsPer-visitor public like/neutral/dislike events.

Demo mode stores these logical collections as JSON documents inside data/grit.sqlite3. Set sqlite = no in config.ini to switch back to MongoDB.

Key guardrails

Access And Visibility

Protected portal access is based on login role plus optional app/access_config.py. Jury category access is based on IDs assigned in each category document. Category deadlines are inherited from the parent cycle. Archived cycles are hidden from active dashboards and retained in archive.
